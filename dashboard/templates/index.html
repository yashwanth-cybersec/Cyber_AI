🛡️ Data Security — How It Works
🔍
File Integrity Monitor (FIM)
At startup, SHA-256 hashes every .py .json .html file in your project. Every poll it re-hashes and checks for: MODIFIED files, DELETED files, and NEW files added unexpectedly. Fires an alert if tampering is detected — protects against malware that modifies your system files.
💾
Automated Backups
Saves timestamped snapshots of your critical data files: memory.json (threat history), audit_log.json (all actions), fim_baseline.json (file hashes), identity_baseline.json (user profiles). Click View → to inspect any backup file's exact contents.
🔐
Auth & Access Control
All user passwords stored as SHA-256 hashes in auth.json — never plain text. Sessions use a per-run random secret key. The Prevention tab requires a separate admin PIN for destructive actions (kill process, block IP). Backup file viewer enforces path sandboxing — you cannot read files outside backups/.
✅ Real-time hash monitoring every poll
✅ Timestamped immutable backups
✅ Path-sandboxed file viewer
✅ Hashed password storage
✅ Admin PIN for prevention actions
